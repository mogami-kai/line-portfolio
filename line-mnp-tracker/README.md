# LINE MNP実績自動集計

携帯ショップのスタッフがLINEグループに投稿する営業実績を自動解析し、スプレッドシートへ集計・月次ランキングを生成するシステムです。

共通アーキテクチャや設計思想はルートの [README](../README.md) を参照してください。

---

## 解決した課題

約60人のスタッフが毎日LINEグループに投稿する営業実績（MNP件数・光回線件数など）を、管理者が週末にまとめて集計していた。毎週末2時間ほどの手作業で、転記ミスや集計タイミングのズレが課題だった。

また、業務グループに雑談も混在するLINE運用だったため、実績報告以外のメッセージを自動でスキップする仕組みも必要だった。

このシステムは `line-daily-report` で作った共通アーキテクチャ（Cloudflare Worker + GAS + 月末締め処理）をそのまま転用し、MNP固有のロジックだけ書き直すことで約1週間で完成させた。

---

## 入力 → 処理 → 出力

**入力（LINEメッセージ）**

```
【1日の店舗実績】
3/18（月）
ヤマダ電機〇〇店
田中
当日MNP 3件/計15件
当日1G光 1件
稼働数 3日
```

**処理**

1. Cloudflare Workerで即時応答 → GASへ非同期転送
2. メッセージIDで重複排除
3. ヘッダー判定で雑談をスキップ（「1日の実績」と「個人獲得」の両方を含むもののみ処理）
4. ルールベースで件数を抽出（全角・半角・表記揺れを正規化してから解析）
5. ルールで解析できなかった場合のみAIでパース
6. スプレッドシートへ登録・月次ランキングを更新

**出力（スプレッドシートに自動登録）**

| 日付 | 店舗 | 担当者 | MNP（当日） | MNP（月累計） | 光 | 稼働数 |
|---|---|---|---|---|---|---|
| 2026/03/18 | ヤマダ電機〇〇店 | 田中 | 3 | 15 | 1 | 3 |

## 難しかった点・工夫

### 表記の揺れへの対応

同じ意味でも書き方が複数ある。単純なキーワード一致では安定しないため、まずNFKCで全角・半角・スペースを正規化してから解析している。

| 意味 | 実際の書き方の例 |
|---|---|
| MNP件数 | `当日MNP 3件`・`mnp:3`・`MNP３`・`ＭＮＰ　３` |
| 光回線 | `当日1G光 1件`・`光1g 1`・`1G光1件` |

### 累計報告と当日報告の混在

スタッフによって「当日MNP 3件」（当日のみ）と「当日MNP 3件/計15件」（月累計付き）の両方の書き方が混在していた。月次ランキングの正確性を担保するため、累計報告がある場合はそちらを優先して集計する設計にした。

### 雑談スキップ

業務グループに雑談も流れる運用だったため、実績報告のみを処理対象に絞る必要があった。ヘッダーに「1日の実績」と「個人獲得」の両方が含まれるメッセージのみ処理対象とし、それ以外はログに記録したうえでスキップする。

## 技術構成

| ファイル | 役割 |
|---|---|
| `webhook.js` | Webhook受信・メッセージ解析・スプレッドシート書き込み・ランキング集計 |
| `management.js` | 月末締め処理・アーカイブ・手動メニュー |
| `worker.js` | Cloudflare Worker（即時200返却 → GASへ非同期転送） |

---

## 制約

- 解析精度はメッセージの書き方に依存する。まったく想定外の形式はスキップになる場合がある
- 実際の店舗名・担当者名は公開していない（サンプルは匿名化済み）
- APIキー・Webhook URL等の秘密情報はGASスクリプトプロパティと環境変数で管理しており、コードに含まれていない
