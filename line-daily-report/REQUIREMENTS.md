# 現場日報 → 集計 → 請求書 自動化 要件定義（v1・GAS版）

> **注**: システム全体の最新要件は、リポジトリ直下の [`REQUIREMENTS.md`（v2）](../REQUIREMENTS.md)（Next.js/Vercel/Supabase/Prisma/LIFF）を参照。本書は GAS（`line-daily-report/`）モジュールの仕様＝**ロジックの移植元**として残置する。


> 実運用ログ（LINEグループの出面日報）と請求書テンプレートから起こした要件。
> 本書・コード・テストの例はすべて**ダミー名**（取引先A／現場X／職人A 等）。実名・実住所・口座等はリポジトリに含めず、スプレッドシート側に入力する。

## 1. 背景・現状

- 6人の施工チームが **LINEグループ**に毎日「出面（でづら）」を投稿。
- LINE Messaging API bot →（Cloudflare Worker）→ **GAS** がパースして `作業日報` シートへ転記。
- パースは **従来ルール**＋フォーマット崩れ時に **AI（Gemini）補助**。登録後、bot がグループへ受信確認を返信。
- 月末24時前後に当月分を Drive へ自動アーカイブ＋当月リセット。
- 既存シート: `月次集計` / `作業日報` / `メッセージ処理ログ` / `経費` / `管理者一覧`。
- ゴール: **取引先ごとの請求書を自動生成**（末締め・請求日＝支払期限）。経費（駐車/燃料）も集計。

## 2. 入力フォーマット（LINE日報）

1メッセージに複数の現場ブロック（空行区切り）が入りうる。

```
[日付（曜日）]            ← 例: 3月10日 / 1/16(火) / 2026/1/16 / 9/25日
[取引先]  [常用|請負]     ← 契約種別は省略されることがある
[現場]
[職人名 …]               ← 各人に半日/夜勤/残業マーカーが付くことがある
[全体マーカー]            ← 末尾に「残業2h」「夜勤」等。その現場の全員に適用
```

### 表記バリエーション（実運用で観測）
- **日付**: `M月D日` / `M/D` / `yyyy/M/D` / 曜日付き / 行頭以外。
- **契約種別**: `常用` / `請負` / `取決`（省略あり）。
- **勤務体系**: 日勤(既定=1) / `半日`(0.5) / `夜勤`(1日) / まれに夜勤0.5。
- **残業**: `残業1` `残1` `1h` `残1h` `OT1` `OT2` `残業2h`。位置は人ごと or 末尾一括。
- **人工**: 0.5 / 0.75 / 1。

## 3. 単価計算ルール（請求の核）

`時給 = 単価 ÷ 8`、残業係数 `1.25`。

| 勤務 | 計算 |
|---|---|
| 1日 | 単価 × 1 |
| 半日 | 単価 × 0.5 |
| 0.75日 | 単価 × 0.75 |
| 残業 Nh | ＋ 単価 ÷ 8 × 1.25 × N |
| 夜勤 | 1日扱い（割増なし。現場により別途） |

- 常用請求額 = Σ(人工 × 単価) ＋ Σ(残業h × 単価/8×1.25)
- 請負 = 案件マスタの契約金額（人工非依存）

## 4. 請求書に直結する4項目とバリデーション（聞き返し）

請求書を壊さないために、**取引先・日付・人工・残業**だけを厳格に検証する。

判定は3値: **ok（そのまま記録）／ confirm（仮記録＋1タップ確認）／ hold（保留＋質問）**。

| 項目 | ok | confirm | hold |
|---|---|---|---|
| 取引先 | マスタ（別名含む）一致 | 近い既存名あり（表記揺れ候補提示） | 一致なし（新規?） |
| 日付 | 実在・直近45日内・未来でない | 未来日 / 45日超過去（月の打ち間違い?） | 解釈不能 / 存在しない日 |
| 人工 | 0.5/0.75/1（0.25刻み・≤1） | >1 / 0.25刻みでない | 0以下 / 非数値 |
| 残業 | 0〜3h | 3〜8h | 負 / 非数値（>8hはconfirm） |
| 職人名 | 記号/数字なし | 名前にマーカー食い込み（例「職人A半日」） | 空 |
| （行間） | — | 同一 取引先×現場×日付 で職人重複 | — |

判定フロー（あなたの設計）:

```
LINE投稿 → パース
 ├ ① ルール判定（上表・機械的）
 │    └ 全項目ok ＆ 取引先一致 → そのまま記録（無確認）
 └ どれか引っかかる → ② AI精査
      ├ AIが自信あり → 仮記録＋1タップ確認（Flex/QuickReply postback）
      └ 自信なし/読めない → 保留＋質問
1タップの回答で確定し、取引先の揺れは別名マスタに学習（次回から無確認）。
```

実装: `report_validate.js`（純粋関数中心・テスト済 `test/report_validate.test.js`）。
- `validateClientField_` / `validateDateField_` / `validateQtyField_` / `validateOtField_` / `validateWorkerName_`
- `validateReportRows_`（重複検出含む）/ `buildAskbackMessage_`（聞き返し文生成）
- `validateReportWithMaster_`（`取引先マスタ`の正式名・別名で解決）

## 5. 名寄せ（表記揺れ吸収）

- `取引先マスタ` の **別名** 列にゆれを登録 → `canonicalClient_` が正式名へ解決。
- 自動検出: 正規化（全角→半角・法人格/空白/中黒除去・小文字化）後の**レーベンシュタイン距離**で近い既存名を候補提示（ローマ字 L/R・末尾ゆれ・漢字1字違い 工/興 等）。
- 確定した揺れは別名へ追記して恒久学習。

## 6. 請求書フォーマット（テンプレ＝設計済スプレッドシート準拠）

取引先ごとに1枚。レイアウト（`invoice_doc.js` の出力をこれに合わせる）:

```
請求書番号  yyyy-NNN(例 2026-001)        請求日  yyyy/MM/dd(末締め)
─ 発行元（請求書設定シート）──────────
[自社名／屋号] / 〒住所 / TEL・Email / 登録番号 T0000000000000 / 担当
─ 宛先 ───────────────────────────
[取引先名] 御中 / [住所]（取引先マスタから）
─ サマリ ─────────────────────────
小計（税抜） / 消費税 / 合計（税込）
税率内訳: 10%対象=対象額・税額 / 8%対象(軽減)=対象額・税額
ご請求金額（税込） / お支払期限 = 請求日（同日）
─ 明細（数式自動計算）────────────────
No | 品目・内容 | 数量 | 単位 | 単価 | 金額 | 税率
  常用: 「{現場} 常用」 数量=人工 単位=人工 単価=入力 金額=数量×単価+残業
  残業: 「{現場} 残業」 数量=h 単価=単価/8×1.25
  請負: 「{案件} 一式」 数量=1
  立替経費(非課税/対象外): 駐車・燃料等
─ フッタ ─────────────────────────
お振込先（請求書設定） / 備考: ※お振込手数料は御社にてご負担…
```

- 単価は **管理者が「単価(入力)」セルに入力** → 金額・小計・消費税・合計が数式で自動。
  （単価は現場ごとに変わるためマスタ固定にしない方針。聞かずに入力式で汎用化）
- 出力: 編集可能 Google スプレッドシート → `.xlsx` 書き出し。

## 7. データモデル（シート）

| シート | 役割 | 主な列 |
|---|---|---|
| 取引先マスタ | 請求の唯一のマスタ | 取引先 / 現場 / 単価 / 住所 / 請求日ルール / 別名 / 備考 |
| 請求書設定 | 自社情報 | 発行元名 / 住所 / TEL / 登録番号 / 振込先 / 消費税率 / 保存フォルダID |
| 作業日報 | 日次ログ | 日付 / 取引先 / 契約種別 / 勤務体系 / 現場 / 職人名 / 人工 / 残業時間 / 確認状態 … |
| 経費 | 立替 | 日付 / 取引先 / 現場 / 職人 / 種別 / 金額 / 請求対象 … |
| 請負案件マスタ(任意) | 一括請負 | 取引先 / 案件名 / 契約金額 / 計上月 … |

## 9. 段階導入

1. **Phase 1**: ルール判定＋「仮記録＋1タップ確認」。4項目の範囲＋マスタ照合だけで大半を無確認 or 1タップ化（AIコスト0）。
2. **Phase 2**: 引っかかった中間だけ AI 精査（近い名前の推定・日付妥当性）。
3. **Phase 3**: 別名・範囲が育ち聞き返しが減衰。請求書は末締めで自動生成。
